================================================================================
                    عملية: تنفيذ البيع (Invoice-Based Sales)
================================================================================

📋 نظرة عامة:
--------------
هذه المرحلة تُحوّل النظام إلى متجر حقيقي.
البيع لا يُحدد السعر — بل يسترجعه من نظام التسعير (Step 2).
✅ عند كل بيع: تُنشأ فاتورة + يُخصم المخزون فوراً.
✅ جميع المبيعات تتم عبر invoices (لا يوجد بيع مباشر بدون فاتورة).
✅ يدعم تتبع العملاء وطرق الدفع.

================================================================================
                        🗂️ الجداول المستخدمة
================================================================================

┌─────────────────────────────────────────────────────────────────────────────┐
│ 1️⃣ customers                                                                │
└─────────────────────────────────────────────────────────────────────────────┘

وظيفته:
تسجيل بيانات العملاء — اختياري (يمكن إنشاء فاتورة بدون عميل).

الحقل              النوع          العلاقة    القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────
id                 BIGINT         PK         AUTO_INCREMENT        المعرف
name               VARCHAR(255)   —          —                     اسم العميل
phone              VARCHAR(20)    —          NULL                  رقم الهاتف
email              VARCHAR(255)   —          NULL                  البريد الإلكتروني
address            TEXT           —          NULL                  العنوان
total_purchases    DECIMAL(10,2)  —          0                     إجمالي المشتريات (يُحدَّث تلقائياً)
is_active          BOOLEAN        —          TRUE                  حالة النشاط
created_at         TIMESTAMP      —          CURRENT_TIMESTAMP     تاريخ التسجيل
updated_at         TIMESTAMP      —          CURRENT_TIMESTAMP     تاريخ التحديث

Constraints:
   UNIQUE (phone)
   INDEX (is_active)


┌─────────────────────────────────────────────────────────────────────────────┐
│ 2️⃣ payment_methods                                                          │
└─────────────────────────────────────────────────────────────────────────────┘

وظيفته:
تعريف طرق الدفع المتاحة في النظام.

الحقل              النوع         العلاقة    القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────
id                 BIGINT        PK         AUTO_INCREMENT        المعرف
name               VARCHAR(50)   —          —                     اسم الطريقة (نقدي / بطاقة / تحويل)
is_active          BOOLEAN       —          TRUE                  حالة النشاط
created_at         TIMESTAMP     —          CURRENT_TIMESTAMP     تاريخ الإنشاء
updated_at         TIMESTAMP     —          CURRENT_TIMESTAMP     تاريخ التحديث

┌─────────────────────────────────────────────────────────────────────────────┐
│ 3️⃣ invoices                                                                 │
└─────────────────────────────────────────────────────────────────────────────┘

وظيفته:
رأس الفاتورة — كل عملية بيع تُنشئ فاتورة.

الحقل              النوع          العلاقة                   القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────────────
id                 BIGINT         PK                        AUTO_INCREMENT        المعرف
invoice_number     VARCHAR(50)    —                         —                     رقم الفاتورة (فريد)
customer_id        BIGINT         FK → customers.id         NULL                  العميل (اختياري)
user_id            BIGINT         FK → users.id             —                     البائع
subtotal           DECIMAL(10,2)  —                         0                     المجموع قبل الخصم
discount           DECIMAL(10,2)  —                         0                     خصم على مستوى الفاتورة
total              DECIMAL(10,2)  —                         0                     الإجمالي النهائي (subtotal - discount)
status             ENUM           —                         'completed'           حالة الفاتورة
notes              TEXT           —                         NULL                  ملاحظات
created_at         TIMESTAMP      —                         CURRENT_TIMESTAMP     تاريخ الإنشاء
updated_at         TIMESTAMP      —                         CURRENT_TIMESTAMP     تاريخ التحديث

ENUM status:
   'completed'  → مكتملة (الحالة الافتراضية)
   'cancelled'  → ملغاة
   'refunded'   → مستردة (بعد اكتمال مرتجع)

Constraints:
   UNIQUE (invoice_number)
   INDEX (customer_id)
   INDEX (status)
   INDEX (created_at)


┌─────────────────────────────────────────────────────────────────────────────┐
│ 4️⃣ invoice_items                                                            │
└─────────────────────────────────────────────────────────────────────────────┘

وظيفته:
تفاصيل المنتجات داخل الفاتورة — سطر لكل منتج.

الحقل              النوع          العلاقة                        القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────────────────
id                 BIGINT         PK                             AUTO_INCREMENT        المعرف
invoice_id         BIGINT         FK → invoices.id               —                     الفاتورة
product_id         BIGINT         FK → products.id               —                     المنتج
sale_type          ENUM           —                              —                     نوع البيع (decant/full/unit_based)
size_id            BIGINT         FK → sizes.id                  NULL                  الحجم (NULL للـ UNIT_BASED)
quantity           DECIMAL(10,2)  —                              —                     الكمية
unit_price         DECIMAL(10,2)  —                              —                     سعر الوحدة
discount           DECIMAL(10,2)  —                              0                     خصم على مستوى السطر
total_price        DECIMAL(10,2)  —                              —                     الإجمالي (quantity × unit_price - discount)
created_at         TIMESTAMP      —                              CURRENT_TIMESTAMP     تاريخ الإنشاء

Constraints:
   FOREIGN KEY (invoice_id) ON DELETE CASCADE
   INDEX (invoice_id)
   INDEX (product_id)

┌─────────────────────────────────────────────────────────────────────────────┐
│ 6️⃣ inventory  (تحديث — من Step 1)                                          │
└─────────────────────────────────────────────────────────────────────────────┘

وظيفته:
يُحدَّث تلقائياً بعد كل بيع لخصم الكمية المباعة.

الحقل المحدَّث:
   quantity = quantity - الكمية_المباعة


================================================================================
                        المراحل والجداول المتأثرة
================================================================================

┌─────────────────────────────────────────────────────────────────────────────┐
│ المرحلة 1: (اختياري) تسجيل / اختيار العميل                                │
└─────────────────────────────────────────────────────────────────────────────┘

🔹 الفاعل: البائع (Seller)
🔹 الشرط: اختياري

📊 الجداول المتأثرة:
--------------------

1️⃣ customers
   - قراءة: البحث عن عميل موجود بالاسم أو الهاتف
   - أو إنشاء: تسجيل عميل جديد


┌─────────────────────────────────────────────────────────────────────────────┐
│ المرحلة 2: إنشاء الفاتورة                                                  │
└─────────────────────────────────────────────────────────────────────────────┘

🔹 الفاعل: البائع (Seller)

📊 الجداول المتأثرة:
--------------------

1️⃣ invoices
   - إنشاء رأس الفاتورة
   - customer_id (اختياري), user_id, invoice_number, status='completed'


┌─────────────────────────────────────────────────────────────────────────────┐
│ المرحلة 3: إضافة منتجات للفاتورة                                          │
└─────────────────────────────────────────────────────────────────────────────┘

🔹 الفاعل: البائع (Seller)

📊 الجداول المتأثرة (قراءة فقط):
--------------------

1️⃣ products
   - قراءة: selling_type / price_tier_id / category_id

2️⃣ inventory
   - قراءة: quantity (التحقق من توفر المخزون)

3️⃣ tier_prices / unit_based_prices
   - قراءة: جلب السعر حسب النوع


┌─────────────────────────────────────────────────────────────────────────────┐
│ المرحلة 4: تسجيل المنتجات في الفاتورة                                     │
└─────────────────────────────────────────────────────────────────────────────┘

🔹 الفاعل: البائع (Seller)
🔹 الشرط: inventory.quantity >= الكمية المطلوبة

📊 الجداول المتأثرة:
--------------------

1️⃣ invoice_items
   - إضافة سطر لكل منتج
   - invoice_id, product_id, sale_type, size_id, quantity, unit_price, total_price

2️⃣ inventory
   - quantity = quantity - الكمية_المباعة (لكل منتج)


┌─────────────────────────────────────────────────────────────────────────────┐
│ المرحلة 5: إتمام الفاتورة والدفع                                          │
└─────────────────────────────────────────────────────────────────────────────┘

🔹 الفاعل: البائع (Seller)

📊 الجداول المتأثرة:
--------------------

1️⃣ invoices
   - تحديث: subtotal, discount, total

2️⃣ payments
   - invoice_id, payment_method_id, amount

3️⃣ customers (إذا كان العميل مسجلاً)
   - total_purchases = total_purchases + invoice.total


================================================================================
                            ملخص الجداول المتأثرة
================================================================================

✅ جداول تُنشأ في هذه المرحلة:
   1. customers        (بيانات العملاء)
   2. payment_methods  (طرق الدفع)
   3. invoices         (رأس الفاتورة)
   4. invoice_items    (تفاصيل الفاتورة)
   5. payments         (سجل الدفع)

✅ جداول تُحدَّث:
   - inventory         (خصم الكمية)
   - customers         (تحديث total_purchases)

✅ جداول تُقرأ فقط:
   - products          (selling_type / price_tier_id)
   - tier_prices       (جلب السعر للـ DECANT/FULL/BOTH)
   - unit_based_prices (جلب السعر للـ UNIT_BASED)
   - sizes             (عرض الأحجام المتاحة)
   - users             (معرف البائع)


================================================================================
                        مخطط تدفق عملية البيع
================================================================================

   (اختياري) اختيار / تسجيل العميل
         ↓
   إنشاء فاتورة جديدة (invoices)
         ↓
   اختيار المنتج → قراءة selling_type
         ↓
   ┌─────────────────────────────────────────┐
   │ DECANT/FULL/BOTH → اختيار size          │
   │ UNIT_BASED       → إدخال الكمية         │
   └─────────────────────────────────────────┘
         ↓
   جلب السعر من tier_prices / unit_based_prices
         ↓
   التحقق من المخزون (inventory.quantity >= المطلوب)
         ↓
   إضافة للفاتورة (invoice_items) + خصم المخزون
         ↓
   (تكرار لإضافة منتجات أخرى)
         ↓
   إتمام الفاتورة (حساب الإجمالي)
         ↓
   تسجيل الدفع (payments)

================================================================================
                            ملاحظات مهمة
================================================================================

⚠️ قواعد العمل:
   1. ✅ لا يمكن البيع إذا كانت الكمية غير كافية في المخزون
   2. ✅ السعر يُسجَّل في invoice_items وقت البيع (snapshot)
   3. ✅ size_id = NULL للمنتجات UNIT_BASED
   4. ✅ خصم المخزون يحدث فوراً عند إضافة المنتج للفاتورة
   5. ✅ جميع المبيعات تتم عبر invoices (لا يوجد بيع بدون فاتورة)

🔐 الصلاحيات:
   - البائع (Seller): إنشاء فواتير
   - المدير (Admin): عرض جميع الفواتير + تقارير

📊 التقارير المتاحة:
   - فواتير اليوم / الأسبوع / الشهر
   - أكثر المنتجات مبيعاً
   - مبيعات كل بائع
   - المخزون المتبقي بعد البيع

🎯 مبدأ Invoice-Only:
   - ✅ كل عملية بيع = فاتورة
   - ✅ الفاتورة يمكن أن تحتوي على منتج واحد أو أكثر
   - ✅ الفاتورة يمكن أن تكون بدون عميل (عميل عابر)
   - ✅ جميع التقارير تعتمد على invoices
