================================================================================
                    عملية: تنفيذ البيع (POS Sale)
================================================================================

📋 نظرة عامة:
--------------
هذه المرحلة تُحوّل النظام إلى متجر حقيقي.
البيع لا يُحدد السعر — بل يسترجعه من نظام التسعير (Step 2).
✅ عند كل بيع: يُسجَّل البيع + يُخصم المخزون فوراً.
✅ يدعم البيع المباشر (POS) والبيع بفاتورة متعددة المنتجات.
✅ يدعم تتبع العملاء وطرق الدفع.

                        🗂️ الجداول المستخدمة

┌─────────────────────────────────────────────────────────────────────────────┐
│ 1️⃣ customers                                                                │
└─────────────────────────────────────────────────────────────────────────────┘

وظيفته:
تسجيل بيانات العملاء — اختياري في POS، إلزامي عند إنشاء فاتورة.

الحقل              النوع          العلاقة    القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────
id                 BIGINT         PK         AUTO_INCREMENT        المعرف
name               VARCHAR(255)   —          —                     اسم العميل
phone              VARCHAR(20)    —          NULL                  رقم الهاتف
email              VARCHAR(255)   —          NULL                  البريد الإلكتروني
address            TEXT           —          NULL                  العنوان
total_purchases    DECIMAL(10,2)  —          0                     إجمالي المشتريات (يُحدَّث تلقائياً)
is_active          BOOLEAN        —          TRUE                  حالة النشاط
created_at         TIMESTAMP      —          CURRENT_TIMESTAMP     تاريخ التسجيل
updated_at         TIMESTAMP      —          CURRENT_TIMESTAMP     تاريخ التحديث

   UNIQUE (phone)
   INDEX (is_active)

وظيفته:
تجميع عدة مبيعات في فاتورة واحدة — رأس الفاتورة.

الحقل              النوع          العلاقة                   القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────────────
id                 BIGINT         PK                        AUTO_INCREMENT        المعرف
invoice_number     VARCHAR(50)    —                         —                     رقم الفاتورة (فريد)

tax                DECIMAL(10,2)  —                         0                     الضريبة
total              DECIMAL(10,2)  —                         0                     الإجمالي النهائي
status             ENUM           —                         'pending'             حالة الفاتورة

created_at         TIMESTAMP      —                         CURRENT_TIMESTAMP     تاريخ الإنشاء
updated_at         TIMESTAMP      —                         CURRENT_TIMESTAMP     تاريخ التحديث

ENUM status:

Constraints:
   UNIQUE (invoice_number)
   INDEX (customer_id)
   INDEX (status)
   INDEX (created_at)

┌─────────────────────────────────────────────────────────────────────────────┐
│ 5️⃣ sales                                                                    │
└─────────────────────────────────────────────────────────────────────────────┘

وظيفته:
تسجيل كل عملية بيع — يدعم البيع المباشر (POS) والبيع عبر فاتورة.

الحقل              النوع          العلاقة             القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────────────
id                 BIGINT         PK                  AUTO_INCREMENT        المعرف
user_id            BIGINT         FK → users.id       —                     البائع
customer_id        BIGINT         FK → customers.id   NULL                  العميل (اختياري)
invoice_id         BIGINT         FK → invoices.id    NULL                  الفاتورة (NULL = بيع مباشر POS)
product_id         BIGINT         FK → products.id    —                     المنتج
sale_type          ENUM           —                   —                     نوع البيع
size_id            BIGINT         FK → sizes.id       NULL                  الحجم (NULL للـ UNIT_BASED)
quantity           DECIMAL(10,2)  —                   —                     الكمية
price              DECIMAL(10,2)  —                   —                     السعر النهائي المسجَّل
created_at         TIMESTAMP      —                   CURRENT_TIMESTAMP     تاريخ البيع

ENUM sale_type:
   'decant'     → بيع بالتقسيم (ml / g)
   'full'       → بيع عبوة كاملة
   'unit_based' → بيع بالقطعة أو الغرام

📌 invoice_id = NULL → بيع مباشر POS بدون فاتورة
📌 invoice_id = X   → البيع مرتبط بفاتورة

┌─────────────────────────────────────────────────────────────────────────────┐
│ المرحلة 2: اختيار المنتج                                                   │
└─────────────────────────────────────────────────────────────────────────────┘

🔹 الفاعل: البائع (Seller)

📊 الجداول المتأثرة (قراءة فقط):
--------------------

1️⃣ products
   - قراءة: selling_type / price_tier_id / category_id

2️⃣ inventory

📌 النظام يحدد تلقائياً:

┌─────────────────────────────────────────────────────────────────────────────┐
│ المرحلة 3: عرض خيارات البيع حسب selling_type                               │
└─────────────────────────────────────────────────────────────────────────────┘

🔹 الفاعل: النظام (Auto)

⚠️ الخيارات حسب النوع:

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
DECANT_ONLY:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   - يعرض قائمة الأحجام المتاحة (sizes)
   - البائع يختار الحجم
   - sale_type = 'decant'

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
FULL_ONLY:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   - يعرض حجم العبوة الكاملة فقط
   - sale_type = 'full'

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
BOTH:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   - يعرض الأحجام + خيار العبوة الكاملة
   - البائع يختار النوع والحجم
   - sale_type = 'decant' أو 'full'

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
UNIT_BASED:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   - البائع يُدخل الكمية مباشرة (عدد قطع أو وزن)
   - لا يوجد size
   - sale_type = 'unit_based'


┌─────────────────────────────────────────────────────────────────────────────┐
│ المرحلة 4: جلب السعر                                                       │
└─────────────────────────────────────────────────────────────────────────────┘

🔹 الفاعل: النظام (Auto)

📊 الجداول المتأثرة (قراءة فقط):
--------------------

للـ DECANT / FULL / BOTH:
1️⃣ tier_prices
   - قراءة: price WHERE tier_id = product.tier AND size_id = selected_size

للـ UNIT_BASED:
1️⃣ unit_based_prices
   - قراءة: price_per_unit WHERE product_id = X AND tier_id = product.tier
   - السعر الإجمالي = price_per_unit × quantity


┌─────────────────────────────────────────────────────────────────────────────┐
│ المرحلة 5: تأكيد البيع وتسجيله                                             │
└─────────────────────────────────────────────────────────────────────────────┘

🔹 الفاعل: البائع (Seller)
🔹 الشرط: inventory.quantity >= الكمية المطلوبة

⚠️ سيناريوان:

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🟢 السيناريو 1: بيع مباشر POS (بدون فاتورة)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1️⃣ sales
   - invoice_id = NULL
   - customer_id = NULL (أو معرف العميل إن وُجد)
   - الحقول المستخدمة:
     • user_id, product_id, sale_type, size_id, quantity, price

2️⃣ payments
   - sale_id = sales.id
   - invoice_id = NULL
   - payment_method_id, amount

3️⃣ inventory
   - quantity = quantity - الكمية_المباعة

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔵 السيناريو 2: بيع بفاتورة (متعدد المنتجات)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1️⃣ invoices
   - إنشاء رأس الفاتورة
   - customer_id, user_id, invoice_number, status='pending'

2️⃣ invoice_items
   - إضافة سطر لكل منتج
   - invoice_id, product_id, sale_type, size_id, quantity, unit_price, total_price

3️⃣ sales
   - إنشاء سجل لكل سطر في الفاتورة
   - invoice_id = invoices.id

4️⃣ invoices
   - تحديث: subtotal, total, status='completed'

5️⃣ payments
   - invoice_id = invoices.id
   - sale_id = NULL
   - payment_method_id, amount

6️⃣ inventory
   - quantity = quantity - الكمية_المباعة (لكل منتج)

7️⃣ customers (إذا كان العميل مسجلاً)
   - total_purchases = total_purchases + invoice.total


================================================================================
                            ملخص الجداول المتأثرة
================================================================================

✅ جداول تُنشأ في هذه المرحلة:
   1. customers        (بيانات العملاء)
   2. payment_methods  (طرق الدفع)
   3. invoices         (رأس الفاتورة)
   4. invoice_items    (تفاصيل الفاتورة)
   5. sales            (سجل البيع)
   6. payments         (سجل الدفع)

✅ جداول تُحدَّث:
   - inventory         (خصم الكمية)
   - customers         (تحديث total_purchases)

✅ جداول تُقرأ فقط:
   - products          (selling_type / price_tier_id)
   - tier_prices       (جلب السعر للـ DECANT/FULL/BOTH)
   - unit_based_prices (جلب السعر للـ UNIT_BASED)
   - sizes             (عرض الأحجام المتاحة)
   - users             (معرف البائع)


================================================================================
                        مخطط تدفق عملية البيع
================================================================================

   (اختياري) اختيار / تسجيل العميل
         ↓
   اختيار المنتج → قراءة selling_type
         ↓
   ┌─────────────────────────────────────────┐
   │ DECANT/FULL/BOTH → اختيار size          │
   │ UNIT_BASED       → إدخال الكمية         │
   └─────────────────────────────────────────┘
         ↓
   جلب السعر من tier_prices / unit_based_prices
         ↓
   التحقق من المخزون (inventory.quantity >= المطلوب)
         ↓
   ┌──────────────────────────────────────────────────────┐
   │ POS مباشر:  sales + payments + inventory             │
   │ بفاتورة:    invoices + invoice_items + sales         │
   │             + payments + inventory + customers       │
   └──────────────────────────────────────────────────────┘


================================================================================
                        🔄 حركة المخزون في كل سيناريو

📦 وشق مسك (500g) — بيع 50g:
   inventory.quantity: 500 → 450

================================================================================
                            ملاحظات مهمة

⚠️ قواعد العمل:
   1. ✅ لا يمكن البيع إذا كانت الكمية غير كافية في المخزون
   2. ✅ السعر يُسجَّل في sales وقت البيع (snapshot) — لا يتغير لاحقاً
   3. ✅ size_id = NULL للمنتجات UNIT_BASED
   4. ✅ خصم المخزون يحدث فوراً عند تأكيد البيع
   5. ✅ sale_type يُحدد كيفية خصم المخزون وجلب السعر
