وظيفته:
ضمان تنفيذ العمليات بشكل آمن (Atomic Operations — ACID).

الحقل              النوع         العلاقة    القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────
id                 BIGINT        PK         AUTO_INCREMENT        المعرف
type               VARCHAR(50)   —          —                     نوع العملية (sale / update / delete)
status             ENUM          —          'pending'             حالة العملية
started_at         TIMESTAMP     —          CURRENT_TIMESTAMP     وقت البداية
completed_at       TIMESTAMP     —          NULL                  وقت الانتهاء

ENUM status:
   'pending'    → جارية
   'committed'  → اكتملت بنجاح
   'failed'     → فشلت

id                 BIGINT         PK                  AUTO_INCREMENT        المعرف
transaction_id     BIGINT         FK → transactions.id —                    العملية المرتبطة

quantity           DECIMAL(10,2)  —                   —                     الكمية المحجوزة

وظيفته:

الحقل              النوع          العلاقة          القيمة الافتراضية    الوصف

📌 ملاحظة: هذا نظام محاسبة بسيط مناسب للـ POS — ليس double-entry accounting

┌─────────────────────────────────────────────────────────────────────────────┐
│ المرحلة 1: تنفيذ بيع آمن (Safe Sale Execution)                             │
└─────────────────────────────────────────────────────────────────────────────┘

🔹 الفاعل: البائع (Seller)

📊 تسلسل العمليات:
--------------------

1️⃣ transactions
   - إنشاء سجل جديد
   - status = 'pending'

2️⃣ inventory_reservations
   - حجز الكمية مؤقتاً
   - status = 'reserved'
   - transaction_id (FK → transactions)

3️⃣ [التحقق من الوصفة والمواد]
   - قراءة recipes + recipe_items
   - التحقق من توفر المواد

4️⃣ [تنفيذ البيع — من Step 3]
   - إنشاء سجل في sales
   - خصم inventory

5️⃣ [تنفيذ الوصفة — من Step 5]
   - إنشاء سجل في production_logs
   - تسجيل material_usage

6️⃣ [حساب التكلفة — من Step 4]
   - إنشاء سجل في sale_costs
   - تحديث batch_inventory (FIFO)

7️⃣ accounting_ledger
   - تسجيل 3 قيود:
     • revenue: سعر البيع
     • cost: إجمالي التكلفة
     • profit: الفرق

8️⃣ inventory_reservations
   - تحديث: status = 'released'

9️⃣ transactions
   - تحديث: status = 'committed'
   - completed_at = NOW()

10️⃣ audit_logs
    - تسجيل العملية كاملة

📌 إذا فشلت أي خطوة:
   - Rollback كامل
   - inventory_reservations: status = 'released'
   - transactions: status = 'failed'
   - system_health_logs: تسجيل الخطأ


┌─────────────────────────────────────────────────────────────────────────────┐
│ المرحلة 2: تسجيل أي تعديل في النظام                                       │
└─────────────────────────────────────────────────────────────────────────────┘

🔹 الفاعل: أي مستخدم

📊 الجداول المتأثرة:
--------------------

1️⃣ audit_logs
   - يتم إنشاء سجل لكل عملية
   - الحقول المستخدمة:
     • user_id (من قام بالعملية)
     • action (create / update / delete)
     • entity_type (product / sale / inventory / recipe / ...)
     • entity_id (معرف العنصر المتأثر)
     • changes (JSON: {before: {...}, after: {...}})

📌 أمثلة:
   تعديل سعر Tier A:
   - action: 'update'
   - entity_type: 'tier_price'
   - changes: {before: {price: 40}, after: {price: 45}}

   حذف وصفة:
   - action: 'delete'
   - entity_type: 'recipe'
   - entity_id: 5

================================================================================
                        مخطط تدفق البيع الآمن الكامل
================================================================================

   البائع يبدأ البيع
         ↓
   transactions: status='pending'
         ↓
   inventory_reservations: status='reserved'
         ↓
   التحقق: مخزون كافٍ؟ + وصفة موجودة؟ + مواد كافية؟
         ↓
   ┌─────────────────────────────────────────┐
   │ ✅ نعم → تنفيذ البيع                    │
   │ ❌ لا  → Rollback + failed              │
   └─────────────────────────────────────────┘
         ↓ (نعم)
   sales + inventory + production_logs + material_usage + sale_costs
         ↓
   accounting_ledger (revenue + cost + profit)
         ↓
   inventory_reservations: status='released'
         ↓
   transactions: status='committed'
         ↓
   audit_logs: تسجيل كامل

================================================================================
                        مثال كامل: بيع آمن لـ Lacoste White 5ml
================================================================================

1. البائع يختار Lacoste White + 5ml
2. transactions: id=1, type='sale', status='pending'
3. inventory_reservations: product_id=1, quantity=5, status='reserved', transaction_id=1
4. التحقق:
   - inventory.quantity = 200 >= 5 ✅
   - recipe موجودة ✅
   - كحول متوفر ✅
5. sales: id=1, product_id=1, sale_type='decant', size_id=3, quantity=1, price=40
6. inventory: quantity = 200 - 5 = 195
7. production_logs: sale_id=1, recipe_id=1, quantity_produced=1
8. material_usage: كحول 1ml + زجاجة 0.1 pcs
9. sale_costs: product_cost=2.5, material_cost=0.6, total_cost=3.1
10. accounting_ledger:
    - revenue: 40
    - cost: 3.1
    - profit: 36.9
11. inventory_reservations: status='released'
12. transactions: status='committed', completed_at=NOW()
13. audit_logs: action='create', entity_type='sale', entity_id=1
