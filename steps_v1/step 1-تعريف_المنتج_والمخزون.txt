📋 نظرة عامة:
--------------
هذه المرحلة هي أساس النظام بالكامل. تُعرَّف فيها وحدات القياس، التصنيفات،
مستويات الجودة (Tiers)، المنتجات، ثم يُسجَّل المخزون الابتدائي.
✅ لا يوجد تسعير أو بيع في هذه المرحلة — فقط تعريف هوية المنتج.

================================================================================
                        🗂️ الجداول المستخدمة
================================================================================

┌─────────────────────────────────────────────────────────────────────────────┐
│ 1️⃣ units                                                                    │
└─────────────────────────────────────────────────────────────────────────────┘

وظيفته:
تعريف وحدات القياس المستخدمة في النظام.

الحقل              النوع        العلاقة    القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────
id                 BIGINT       PK         AUTO_INCREMENT        المعرف
name               VARCHAR(50)  —          —                     اسم الوحدة (مليلتر / غرام / قطعة)
symbol             VARCHAR(10)  —          —                     الرمز (ml / g / pcs)
created_at         TIMESTAMP    —          CURRENT_TIMESTAMP     تاريخ الإنشاء
updated_at         TIMESTAMP    —          CURRENT_TIMESTAMP     تاريخ التحديث

Constraints:
   UNIQUE (symbol)


┌─────────────────────────────────────────────────────────────────────────────┐
│ 2️⃣ categories                                                               │
└─────────────────────────────────────────────────────────────────────────────┘

وظيفته:
تصنيف المنتجات وربط كل تصنيف بوحدة قياس محددة.

الحقل              النوع         العلاقة         القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────
id                 BIGINT        PK              AUTO_INCREMENT        المعرف
name               VARCHAR(255)  —               —                     اسم التصنيف (عطور زيتية / بخور / وشق)
unit_id            BIGINT        FK → units.id   —                     الوحدة المرتبطة بهذا التصنيف
created_at         TIMESTAMP     —               CURRENT_TIMESTAMP     تاريخ الإنشاء
updated_at         TIMESTAMP     —               CURRENT_TIMESTAMP     تاريخ التحديث


┌─────────────────────────────────────────────────────────────────────────────┐
│ 3️⃣ price_tiers                                                              │
└─────────────────────────────────────────────────────────────────────────────┘

وظيفته:
تصنيف المنتجات حسب مستوى الجودة، يُستخدم لاحقاً في نظام التسعير.

الحقل              النوع         العلاقة    القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────
id                 BIGINT        PK         AUTO_INCREMENT        المعرف
name               VARCHAR(10)   —          —                     اسم التير (A / B / C)
description        TEXT          —          NULL                  وصف المستوى
created_at         TIMESTAMP     —          CURRENT_TIMESTAMP     تاريخ الإنشاء
updated_at         TIMESTAMP     —          CURRENT_TIMESTAMP     تاريخ التحديث

Constraints:
   UNIQUE (name)


┌─────────────────────────────────────────────────────────────────────────────┐
│ 4️⃣ products                                                                 │
└─────────────────────────────────────────────────────────────────────────────┘

وظيفته:
تعريف المنتج الكامل مع تحديد طريقة بيعه.

الحقل              النوع         العلاقة                  القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────────────
id                 BIGINT        PK                       AUTO_INCREMENT        المعرف
name               VARCHAR(255)  —                        —                     اسم المنتج
category_id        BIGINT        FK → categories.id       —                     التصنيف
price_tier_id      BIGINT        FK → price_tiers.id      —                     مستوى الجودة
selling_type       ENUM          —                        —                     نوع البيع (انظر القيم أدناه)
created_at         TIMESTAMP     —                        CURRENT_TIMESTAMP     تاريخ الإنشاء
updated_at         TIMESTAMP     —                        CURRENT_TIMESTAMP     تاريخ التحديث

ENUM selling_type:
   'DECANT_ONLY'  → بيع بالتقسيم فقط (ml / g)
   'FULL_ONLY'    → بيع عبوة كاملة فقط
   'BOTH'         → يدعم التقسيم والعبوة الكاملة معاً
   'UNIT_BASED'   → بيع بالقطعة أو الغرام (بخور / وشق)


┌─────────────────────────────────────────────────────────────────────────────┐
│ 5️⃣ inventory                                                                │
└─────────────────────────────────────────────────────────────────────────────┘

وظيفته:
تسجيل الكمية الحالية لكل منتج في المخزون.

الحقل              النوع          العلاقة             القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────────
id                 BIGINT         PK                  AUTO_INCREMENT        المعرف
product_id         BIGINT         FK → products.id    —                     المنتج
quantity           DECIMAL(10,2)  —                   0                     الكمية المتوفرة
unit_id            BIGINT         FK → units.id       —                     الوحدة (للعرض السريع)
created_at         TIMESTAMP      —                   CURRENT_TIMESTAMP     تاريخ الإنشاء
updated_at         TIMESTAMP      —                   CURRENT_TIMESTAMP     تاريخ التحديث

Constraints:
   UNIQUE (product_id)

📌 ملاحظة: unit_id هنا للعرض السريع، الوحدة الأصلية تُستخرج من product → category → unit


┌─────────────────────────────────────────────────────────────────────────────┐
│ 6️⃣ product_images                                                           │
└─────────────────────────────────────────────────────────────────────────────┘

وظيفته:
تخزين صور المنتجات — كل منتج يمكن أن يكون له أكثر من صورة.

الحقل              النوع          العلاقة             القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────────
id                 BIGINT         PK                  AUTO_INCREMENT        المعرف
product_id         BIGINT         FK → products.id    —                     المنتج
image_path         VARCHAR(255)   —                   —                     مسار الصورة
is_primary         BOOLEAN        —                   FALSE                 هل هي الصورة الرئيسية؟
order              INT            —                   0                     ترتيب العرض
created_at         TIMESTAMP      —                   CURRENT_TIMESTAMP     تاريخ الإنشاء

   INDEX (product_id)
   INDEX (is_primary)

┌─────────────────────────────────────────────────────────────────────────────┐
│ المرحلة 1: إنشاء وحدات القياس                                              │
└─────────────────────────────────────────────────────────────────────────────┘

🔹 الفاعل: المدير (Admin)

📊 الجداول المتأثرة:
--------------------

1️⃣ units
   - يتم إنشاء سجل لكل وحدة
   - الحقول المستخدمة:
     • name (مليلتر / غرام / قطعة)
     • symbol (ml / g / pcs)

📌 أمثلة:
   ml  → للعطور
   g   → للوشق
   pcs → للبخور


┌─────────────────────────────────────────────────────────────────────────────┐
│ المرحلة 2: إنشاء التصنيفات                                                 │
└─────────────────────────────────────────────────────────────────────────────┘

🔹 الفاعل: المدير (Admin)

📊 الجداول المتأثرة:
--------------------

1️⃣ categories
   - يتم إنشاء سجل لكل تصنيف مع ربطه بوحدة
   - الحقول المستخدمة:
     • name (عطور زيتية / بخور / وشق)
     • unit_id (FK → units)

📌 أمثلة:
   عطور زيتية → unit: ml
   بخور        → unit: pcs
   وشق         → unit: g


┌─────────────────────────────────────────────────────────────────────────────┐
│ المرحلة 3: إنشاء مستويات الجودة (Tiers)                                    │
└─────────────────────────────────────────────────────────────────────────────┘

🔹 الفاعل: المدير (Admin)

📊 الجداول المتأثرة:
--------------------

1️⃣ price_tiers
   - يتم إنشاء سجل لكل مستوى
   - الحقول المستخدمة:
     • name (A / B / C)
     • description (اقتصادي / متوسط / فاخر)

📌 أمثلة:
   A → اقتصادي
   B → متوسط
   C → فاخر


┌─────────────────────────────────────────────────────────────────────────────┐
│ المرحلة 4: إنشاء المنتج                                                    │
└─────────────────────────────────────────────────────────────────────────────┘

🔹 الفاعل: المدير (Admin)

📊 الجداول المتأثرة:
--------------------

1️⃣ products
   - يتم إنشاء سجل لكل منتج
   - الحقول المستخدمة:
     • name (Lacoste White / بخور عود / وشق مسك)
     • category_id (FK → categories)
     • price_tier_id (FK → price_tiers)
     • selling_type (DECANT_ONLY / FULL_ONLY / BOTH / UNIT_BASED)

📌 أمثلة:
   Lacoste White  → category: عطور زيتية | tier: A | selling_type: DECANT_ONLY
   Dior Sauvage   → category: عطور زيتية | tier: C | selling_type: BOTH
   بخور عود       → category: بخور       | tier: B | selling_type: UNIT_BASED
   وشق مسك        → category: وشق        | tier: A | selling_type: UNIT_BASED


┌─────────────────────────────────────────────────────────────────────────────┐
│ المرحلة 5: تسجيل المخزون الابتدائي                                         │
└─────────────────────────────────────────────────────────────────────────────┘

🔹 الفاعل: المدير / أمين المخزن

📊 الجداول المتأثرة:
--------------------

1️⃣ inventory
   - يتم إنشاء سجل لكل منتج (UNIQUE product_id)
   - الحقول المستخدمة:
     • product_id (FK → products)
     • quantity (الكمية الابتدائية)
     • unit_id (FK → units)

📌 أمثلة:
   Lacoste White  → quantity: 200  | unit: ml
   بخور عود       → quantity: 100  | unit: pcs
   وشق مسك        → quantity: 500  | unit: g


┌─────────────────────────────────────────────────────────────────────────────┐
│ المرحلة 6: إضافة صور المنتج                                                │
└─────────────────────────────────────────────────────────────────────────────┘

🔹 الفاعل: المدير (Admin)
🔹 الشرط: المنتج يجب أن يكون موجوداً

📊 الجداول المتأثرة:
--------------------

1️⃣ product_images
   - يتم إنشاء سجل لكل صورة
   - الحقول المستخدمة:
     • product_id (FK → products)
     • image_path (مسار الصورة)
     • is_primary (TRUE لصورة واحدة فقط)
     • order (ترتيب العرض)

📌 قاعدة: كل منتج له صورة رئيسية واحدة فقط (is_primary = TRUE)

================================================================================
                        🔄 تأثير selling_type على باقي النظام
================================================================================

📦 DECANT_ONLY:
   - يُباع بالتقسيم فقط (ml أو g)
   - يحتاج sizes في نظام التسعير (Step 2)
   - المخزون يُخصم بالكمية المباعة

📦 FULL_ONLY:
   - يُباع كعبوة كاملة فقط
   - يحتاج size خاص بالعبوة الكاملة
   - المخزون يُخصم بالعبوة كاملة

📦 BOTH:
   - يدعم التقسيم والعبوة الكاملة
   - يحتاج sizes متعددة
   - المخزون يُخصم حسب نوع البيع

📦 UNIT_BASED:
   - يُباع بالقطعة أو الغرام
   - لا يحتاج sizes (نقطة قرار في Step 2)
   - المخزون يُخصم بعدد القطع أو الوزن
