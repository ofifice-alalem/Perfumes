================================================================================
                    عملية: تتبع حركات المخزون (Inventory Movements)
================================================================================

📋 نظرة عامة:
--------------
هذا الجدول هو السجل الموحد لكل حركة تحدث في المخزون من أي مصدر.
كل عملية تُغيّر المخزون (بيع / شراء / فاقد / مرتجع / تعديل) تُسجَّل هنا تلقائياً.
✅ يُجيب على سؤال واحد: "ما الذي حدث للمخزون؟ متى؟ ولماذا؟"

================================================================================
                        🗂️ الجداول المستخدمة
================================================================================

┌─────────────────────────────────────────────────────────────────────────────┐
│ 1️⃣ inventory_movements                                                      │
└─────────────────────────────────────────────────────────────────────────────┘

وظيفته:
سجل موحد لكل حركات المخزون — يتغذى تلقائياً من جميع العمليات.

الحقل              النوع          العلاقة             القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────────
id                 BIGINT         PK                  AUTO_INCREMENT        المعرف
product_id         BIGINT         FK → products.id    —                     المنتج
movement_type      ENUM           —                   —                     نوع الحركة (انظر القيم أدناه)
quantity           DECIMAL(10,2)  —                   —                     الكمية (موجب للإضافة / سالب للخصم)
quantity_before    DECIMAL(10,2)  —                   —                     الكمية قبل الحركة
quantity_after     DECIMAL(10,2)  —                   —                     الكمية بعد الحركة
reference_type     VARCHAR(50)    —                   NULL                  نوع المرجع (انظر القيم أدناه)
reference_id       BIGINT         —                   NULL                  معرف المرجع
user_id            BIGINT         FK → users.id       —                     المستخدم الذي نفّذ الحركة
notes              TEXT           —                   NULL                  ملاحظات
created_at         TIMESTAMP      —                   CURRENT_TIMESTAMP     تاريخ الحركة

ENUM movement_type:
   'in'          → إضافة مخزون
   'out'         → خصم مخزون
   'adjustment'  → تعديل يدوي
   'transfer'    → نقل (مستقبلي)

reference_type القيم الممكنة:
   'invoice_item' → مرتبط بـ invoice_items.id
   'purchase'     → مرتبط بـ purchase_batches.id
   'loss'         → مرتبط بـ inventory_losses.id
   'return'       → مرتبط بـ returns.id
   NULL           → تعديل يدوي بدون مرجع

Constraints:
   INDEX (product_id)
   INDEX (movement_type)
   INDEX (reference_type, reference_id)
   INDEX (created_at)

                            ملخص الجداول المتأثرة
================================================================================

✅ جداول تُنشأ في هذه المرحلة:
   1. inventory_movements

✅ جداول تُغذّي inventory_movements (من مراحل أخرى):
   - invoice_items      (Step 3) → movement_type: 'out'
   - purchase_batches   (Step 4) → movement_type: 'in'
   - inventory_losses   (Step 4) → movement_type: 'out'
   - returns            (Step 8) → movement_type: 'in'
   - تعديل يدوي        (Admin)  → movement_type: 'adjustment'


================================================================================
                        مخطط تدفق الحركات
================================================================================

   بيع (Step 3)        → inventory_movements (out, ref: invoice_item)
   شراء (Step 4)       → inventory_movements (in,  ref: purchase)
   فاقد (Step 4)       → inventory_movements (out, ref: loss)
   مرتجع (Step 8)      → inventory_movements (in,  ref: return)
   تعديل يدوي (Admin)  → inventory_movements (adjustment, ref: NULL)
                                    ↓
                          inventory.quantity (يُحدَّث دائماً)


================================================================================
                        الفرق بين inventory و inventory_movements
================================================================================

┌──────────────────────────────────────────────────────────────────────────┐
│ inventory                                                                │
│ - يحتوي الكمية الحالية فقط                                              │
│ - سجل واحد لكل منتج                                                     │
│ - يُجيب على: "كم المتوفر الآن؟"                                         │
└──────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────┐
│ inventory_movements                                                      │
│ - يحتوي تاريخ كل حركة                                                   │
│ - سجلات متعددة لكل منتج                                                 │
│ - يُجيب على: "ماذا حدث؟ متى؟ ولماذا؟"                                  │
└──────────────────────────────────────────────────────────────────────────┘


================================================================================
                            ملاحظات مهمة
================================================================================

⚠️ قواعد العمل:
   1. ✅ كل تغيير في inventory يجب أن يُسجَّل في inventory_movements
   2. ✅ quantity_before + quantity = quantity_after دائماً
   3. ✅ reference_type + reference_id يربطان الحركة بمصدرها
   4. ✅ التعديل اليدوي يحتاج notes إلزامية

🔐 الصلاحيات:
   - النظام: إنشاء تلقائي عند كل عملية
   - المدير: التعديل اليدوي فقط

📊 التقارير المتاحة:
   - تاريخ حركات كل منتج
   - إجمالي المبيعات من المخزون
   - إجمالي المشتريات
   - إجمالي الفاقد
   - تقرير الجرد (مقارنة inventory مع movements)
