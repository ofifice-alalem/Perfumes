📋 الفلسفة:
-----------
بساطة Hamza_Perfumes + دعم كامل لجميع أنواع المنتجات + مخزون.
لا تعقيد، لا جداول زائدة، كل جدول له وظيفة واحدة واضحة.

📦 أنواع المنتجات المدعومة:
   - عطور زيتية   → تُباع بالـ ml  | السعر من التير (A/B/C)
   - عطور أصلية   → تُباع بالـ ml أو عبوة كاملة | سعر خاص لكل منتج
   - بخور          → يُباع بالقطعة (pcs) | سعر خاص لكل منتج
   - وشق           → يُباع بالغرام (g)   | سعر خاص لكل منتج

📋 نظام الفواتير:
   كل عملية بيع = فاتورة (invoice) تحتوي على سطر أو أكثر (invoice_items).
   الفاتورة تتتبع حالة الدفع، والسطر يحفظ السعر وقت البيع (snapshot).

📌 الوظيفة:
   جدول الأسعار المركزي للعطور الزيتية.
   يربط كل تير بكل حجم بسعر محدد (regular و vip).
   تغيير سعر واحد هنا يؤثر على جميع العطور في نفس التير تلقائياً —
   لا حاجة لتعديل كل عطر على حدة.

الحقل              النوع           العلاقة                  القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────────────
id                 BIGINT          PK                       AUTO_INCREMENT        المعرف
tier_id            BIGINT          FK → price_tiers.id      —                     التير

price_regular      DECIMAL(10,2)   —                        —                     سعر العميل العادي
price_vip          DECIMAL(10,2)   —                        —                     سعر عميل VIP
created_at         TIMESTAMP       —                        CURRENT_TIMESTAMP     تاريخ الإنشاء
updated_at         TIMESTAMP       —                        CURRENT_TIMESTAMP     تاريخ التحديث

Constraints:
   UNIQUE (tier_id, size_id)

📌 منطق جلب السعر عند البيع:
   SELECT price_regular / price_vip
   FROM tier_prices
   WHERE tier_id = product.price_tier_id
     AND size_id = ?

📌 أمثلة:
   Tier A + 1ml  → regular: 5  | vip: 4
   Tier A + 3ml  → regular: 12 | vip: 10
   Tier A + 5ml  → regular: 18 | vip: 15
   Tier B + 1ml  → regular: 7  | vip: 6
   Tier C + 5ml  → regular: 30 | vip: 25

│ 7️⃣  product_prices                                                          │
└─────────────────────────────────────────────────────────────────────────────┘

📌 الوظيفة:
   جدول الأسعار الخاصة للمنتجات من نوع 'unit_priced'
   (عطور أصلية، بخور، وشق).
   كل منتج له سعر وحدة خاص به (سعر الـ ml أو القطعة أو الغرام).
   للعطور الأصلية: يوجد أيضاً سعر العبوة الكاملة كخيار إضافي.

الحقل                  النوع           العلاقة              القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────────────
id                     BIGINT          PK                   AUTO_INCREMENT        المعرف
product_id             BIGINT          FK → products.id     —                     المنتج
price_per_unit_regular DECIMAL(10,2)   —                    —                     سعر الوحدة (عادي)
price_per_unit_vip     DECIMAL(10,2)   —                    —                     سعر الوحدة (VIP)
full_bottle_regular    DECIMAL(10,2)   —                    NULL                  سعر العبوة كاملة (عادي)
full_bottle_vip        DECIMAL(10,2)   —                    NULL                  سعر العبوة كاملة (VIP)
created_at             TIMESTAMP       —                    CURRENT_TIMESTAMP     تاريخ الإنشاء
updated_at             TIMESTAMP       —                    CURRENT_TIMESTAMP     تاريخ التحديث

Constraints:
   UNIQUE (product_id)

📌 قاعدة:
   - full_bottle_regular / full_bottle_vip → NULL للبخور والوشق (لا عبوة كاملة)
   - للعطور الأصلية: يمكن ملؤهم أو تركهم NULL إذا لا يُباع بالعبوة

📌 منطق جلب السعر عند البيع:
   ┌─ بيع بالوحدة (ml / pcs / g):
   │   السعر = price_per_unit_regular أو price_per_unit_vip × الكمية
   │
   └─ بيع عبوة كاملة (للعطور الأصلية فقط):
       السعر = full_bottle_regular أو full_bottle_vip (سعر ثابت)

📌 أمثلة:
   Dior Sauvage (200ml) → per_unit_regular: 3.5 | per_unit_vip: 3
                           full_bottle_regular: 500 | full_bottle_vip: 450
   بخور عود             → per_unit_regular: 5   | per_unit_vip: 4
                           full_bottle: NULL (لا يُباع بعبوة)
   وشق مسك              → per_unit_regular: 2   | per_unit_vip: 1.5
                           full_bottle: NULL (لا يُباع بعبوة)


┌─────────────────────────────────────────────────────────────────────────────┐
│ 8️⃣  invoices                                                                │
└─────────────────────────────────────────────────────────────────────────────┘

📌 الوظيفة:
   رأس كل عملية بيع — كل بيعة تبدأ بإنشاء فاتورة.
   الفاتورة تجمع معلومات البائع ونوع العميل وحالة الدفع.
   الإجمالي يُحسب من مجموع أسطر invoice_items.
   تفاصيل الدفع (الوسيلة والمبلغ) تُسجَّل في جدول payments المنفصل.

الحقل              النوع           العلاقة              القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────────────
id                 BIGINT          PK                   AUTO_INCREMENT        المعرف
user_id            BIGINT          FK → users.id        —                     البائع
customer_id        BIGINT          FK → customers.id    NULL                  العميل (NULL = عابر)
customer_type      ENUM            —                    'regular'             نوع العميل
total              DECIMAL(10,2)   —                    0                     الإجمالي (يُحسب من الأسطر)
paid_amount        DECIMAL(10,2)   —                    0                     إجمالي المدفوع (يُحسب من payments)
due_amount         DECIMAL(10,2)   —                    0                     المتبقي = total - paid_amount
payment_status     ENUM            —                    'unpaid'              حالة الدفع
notes              TEXT            —                    NULL                  ملاحظات
created_at         TIMESTAMP       —                    CURRENT_TIMESTAMP     تاريخ الفاتورة
updated_at         TIMESTAMP       —                    CURRENT_TIMESTAMP     تاريخ التحديث

ENUM customer_type:
   'regular' → عميل عادي
   'vip'     → عميل VIP

ENUM payment_status:
   'unpaid'  → لم يُدفع شيء بعد
   'partial' → دُفع جزء من المبلغ
   'paid'    → مدفوعة بالكامل

📌 قاعدة total:
   total = SUM(invoice_items.line_total)
   يُحدَّث تلقائياً عند إضافة أو حذف أي سطر

📌 قاعدة paid_amount و due_amount:
   paid_amount = SUM(payments.amount) لهذه الفاتورة
   due_amount  = total - paid_amount
   payment_status يُحدَّث تلقائياً:
     paid_amount = 0           → 'unpaid'
     0 < paid_amount < total   → 'partial'
     paid_amount >= total      → 'paid'

📌 قاعدة الدين:
   customer_id = NULL              → عميل عابر، لا دين
   customer_id موجود + due_amount > 0 → customers.total_debt += due_amount
   لا يمكن ترك due_amount > 0 إذا customer_id = NULL

Constraints:
   INDEX (user_id)
   INDEX (customer_id)
   INDEX (payment_status)
   INDEX (created_at)


┌─────────────────────────────────────────────────────────────────────────────┐
│ 9️⃣  invoice_items                                                           │
└─────────────────────────────────────────────────────────────────────────────┘

📌 الوظيفة:
   سطر واحد لكل منتج داخل الفاتورة.
   هنا يُحفظ السعر وقت البيع (snapshot) — حتى لو تغيرت الأسعار
   لاحقاً، السجل التاريخي يبقى صحيحاً.
   هنا أيضاً يُحدَّد نوع البيع: هل بالحجم أم بالعبوة الكاملة؟
   وعند إنشاء السطر: يُخصم المخزون فوراً للمنتجات unit_priced.

الحقل              النوع           العلاقة                  القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────────────
id                 BIGINT          PK                       AUTO_INCREMENT        المعرف
invoice_id         BIGINT          FK → invoices.id         —                     الفاتورة
product_id         BIGINT          FK → products.id         —                     المنتج
size_id            BIGINT          FK → sizes.id            NULL                  الحجم (NULL للعبوة الكاملة)
sale_type          ENUM            —                        —                     نوع البيع
quantity           DECIMAL(10,2)   —                        1                     الكمية
unit_price         DECIMAL(10,2)   —                        —                     سعر الوحدة وقت البيع (snapshot)
line_total         DECIMAL(10,2)   —                        —                     الإجمالي = unit_price × quantity
created_at         TIMESTAMP       —                        CURRENT_TIMESTAMP     تاريخ الإنشاء

ENUM sale_type:
   'tier_decant'   → عطر زيتي مباع بالحجم (ml)
                     السعر من tier_prices | خصم المخزون: stock -= size.value
   'unit_decant'   → عطر أصلي / بخور / وشق مباع بالوحدة
                     السعر = product_prices.price_per_unit × quantity
                     خصم المخزون: stock -= (size.value × quantity)
   'full_bottle'   → عطر أصلي مباع بالعبوة الكاملة
                     السعر = product_prices.full_bottle_regular/vip
                     خصم المخزون: stock -= stock الحالي كله

📌 قاعدة size_id:
   - sale_type = 'tier_decant'  → size_id مطلوب (يحدد الحجم والسعر)
   - sale_type = 'unit_decant'  → size_id مطلوب (يحدد الكمية المخصومة)
   - sale_type = 'full_bottle'  → size_id = NULL

📌 قاعدة quantity:
   - sale_type = 'tier_decant'  → quantity = 1 (الحجم في size_id)
   - sale_type = 'unit_decant'  → quantity = الكمية الفعلية (مثلاً: 15ml أو 10pcs)
   - sale_type = 'full_bottle'  → quantity = 1

📌 قاعدة unit_price (snapshot):
   - 'tier_decant'  → tier_prices.price_regular أو price_vip
   - 'unit_decant'  → product_prices.price_per_unit_regular أو vip
   - 'full_bottle'  → product_prices.full_bottle_regular أو vip
   يُحدَّد حسب invoices.customer_type وقت البيع ولا يتغير بعدها

📌 قاعدة خصم المخزون:
   - 'tier_decant'  → products.stock -= sizes.value
   - 'unit_decant'  → products.stock -= (sizes.value × quantity)
   - 'full_bottle'  → products.stock -= products.stock (تصفير)

Constraints:
   FOREIGN KEY (invoice_id) ON DELETE CASCADE
   INDEX (invoice_id)
   INDEX (product_id)

   تعريف وسائل الدفع المتاحة في النظام.
   جدول مستقل يسمح بإضافة وسائل جديدة بدون تعديل في الكود.
   مثال: نقدي، بطاقة، تحويل بنكي.

الحقل          النوع           القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────
id             BIGINT          AUTO_INCREMENT        المعرف
name           VARCHAR(50)     —                     اسم الوسيلة
is_active      BOOLEAN         TRUE                  هل هي متاحة حالياً؟
created_at     TIMESTAMP       CURRENT_TIMESTAMP     تاريخ الإنشاء
updated_at     TIMESTAMP       CURRENT_TIMESTAMP     تاريخ التحديث

Constraints:
   UNIQUE (name)

📌 أمثلة:
   نقدي          → is_active: TRUE
   بطاقة         → is_active: TRUE
   تحويل بنكي    → is_active: TRUE


┌─────────────────────────────────────────────────────────────────────────────┐
│ 1️⃣1️⃣ payments                                                               │
└─────────────────────────────────────────────────────────────────────────────┘

📌 الوظيفة:
   تسجيل كل دفعة مرتبطة بفاتورة.
   فاتورة واحدة يمكن أن يكون لها عدة دفعات —
   سواء على مراحل زمنية أو بوسائل دفع مختلفة في نفس الوقت.
   مثال: فاتورة 100 دينار → 60 نقدي + 40 بطاقة.

الحقل                النوع           العلاقة                    القيمة الافتراضية    الوصف
──────────────────────────────────────────────────────────────────────────────────────────
id                   BIGINT          PK                         AUTO_INCREMENT        المعرف
invoice_id           BIGINT          FK → invoices.id           —                     الفاتورة
payment_method_id    BIGINT          FK → payment_methods.id    —                     وسيلة الدفع
amount               DECIMAL(10,2)   —                          —                     المبلغ المدفوع
notes                VARCHAR(255)    —                          NULL                  ملاحظة (رقم مرجعي مثلاً)
created_at           TIMESTAMP       —                          CURRENT_TIMESTAMP     تاريخ الدفعة

Constraints:
   INDEX (invoice_id)
   INDEX (created_at)

📌 عند إضافة دفعة:
   1. INSERT في payments
   2. UPDATE invoices:
        paid_amount = SUM(payments.amount) WHERE invoice_id = ?
        due_amount  = total - paid_amount
        payment_status يُحدَّث حسب القاعدة أعلاه

📌 مثال — فاتورة 100 دينار تُدفع بوسيلتين:
   INSERT payments (invoice_id=1, payment_method_id=1, amount=60)  ← نقدي
   INSERT payments (invoice_id=1, payment_method_id=2, amount=40)  ← بطاقة
   ─────────────────────────────────────────────────────────────────
   invoices: paid_amount=100 | due_amount=0 | payment_status='paid'

                        🔄 مخطط العلاقات الكاملة

   users
     └── invoices.user_id

   customers
     └── invoices.customer_id

   products (selling_type + stock)
     ├── product_prices.product_id  [unit_priced فقط]
     └── invoice_items.product_id

   invoices  (رأس الفاتورة: بائع + عميل + إجمالي + حالة دفع)
     ├── invoice_items.invoice_id   (أسطر الفاتورة: منتج + سعر snapshot)
     └── payments.invoice_id        (دفعات الفاتورة: وسيلة + مبلغ)

   payment_methods
     └── payments.payment_method_id

   tier_prices     ← سعر العطور الزيتية   (tier + size → price)
   product_prices  ← سعر العطور الأصلية / البخور / الوشق


================================================================================
                        📊 منطق التسعير حسب النوع
================================================================================

┌─────────────────────────────────────────────────────────────────────────────┐
│ عطر زيتي — sale_type: tier_decant                                          │
└─────────────────────────────────────────────────────────────────────────────┘

   البائع يختار: عطر + حجم + نوع عميل
         ↓
   إنشاء invoice (أو إضافة لفاتورة مفتوحة)
         ↓
   جلب tier_id من products
   SELECT price_regular/vip FROM tier_prices
   WHERE tier_id = ? AND size_id = ?
         ↓
   إنشاء invoice_item:
     sale_type = 'tier_decant' | size_id = ? | quantity = 1
     unit_price = السعر المجلوب | line_total = unit_price
   خصم المخزون: products.stock -= sizes.value


┌─────────────────────────────────────────────────────────────────────────────┐
│ عطر أصلي / بخور / وشق — sale_type: unit_decant                            │
└─────────────────────────────────────────────────────────────────────────────┘

   البائع يختار: منتج + كمية + نوع عميل
         ↓
   إنشاء invoice
         ↓
   SELECT price_per_unit_regular/vip FROM product_prices
   WHERE product_id = ?
         ↓
   إنشاء invoice_item:
     sale_type = 'unit_decant' | size_id = ? | quantity = الكمية
     unit_price = price_per_unit | line_total = unit_price × quantity
   خصم المخزون: products.stock -= (size.value × quantity)


┌─────────────────────────────────────────────────────────────────────────────┐
│ عطر أصلي — sale_type: full_bottle                                          │
└─────────────────────────────────────────────────────────────────────────────┘

   البائع يختار: منتج + "عبوة كاملة" + نوع عميل
         ↓
   إنشاء invoice
         ↓
   SELECT full_bottle_regular/vip FROM product_prices
   WHERE product_id = ?
         ↓
   إنشاء invoice_item:
     sale_type = 'full_bottle' | size_id = NULL | quantity = 1
     unit_price = full_bottle_price | line_total = unit_price
   خصم المخزون: products.stock = 0

9.  ✅ unit_price يُسجَّل في invoice_items وقت البيع ولا يتغير (snapshot)
10. ✅ تغيير tier_prices يؤثر على جميع عطور نفس التير (للمبيعات الجديدة فقط)
11. ✅ invoices.total = SUM(invoice_items.line_total) يُحدَّث عند كل تغيير في الأسطر
12. ✅ حذف invoice → يحذف جميع invoice_items تلقائياً (CASCADE)
13. ✅ paid_amount = SUM(payments.amount) يُحدَّث عند كل دفعة جديدة
14. ✅ لا يمكن أن يتجاوز paid_amount قيمة total
15. ✅ إذا due_amount > 0 و customer_id = NULL → خطأ (لا دين على عميل عابر)
16. ✅ customers.total_debt يُحدَّث تلقائياً عند كل فاتورة أو دفعة
17. ✅ customers.total_purchases يُحدَّث عند كل فاتورة مكتملة الدفع
